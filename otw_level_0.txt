Level 0 → Level 1

Objective
Gain access to the next level by retrieving the password stored in a file within the home directory.

Key Concept
Basic Linux enumeration and file reading.

This level focuses on:
- Identifying accessible files in a directory
- Extracting sensitive information from readable files

Commands Used
ls
cat readme

Step-by-Step Breakdown

1. Enumerate directory contents

ls

- Listed all files in the current directory
- Discovered a file named "readme"
- This indicates potential storage of useful or sensitive information

2. Inspect file contents

cat readme

- Displayed the contents of the file directly in the terminal
- Retrieved the password required for the next level

Analysis / Takeaways

- Enumeration is always the first step in any system exploration or attack
- Files with generic names (e.g., "readme") can contain sensitive data
- Misplaced or unprotected files are a common security weakness

What Could Go Wrong?

- If permissions were restricted, cat would fail → would need alternative methods (e.g., checking permissions with ls -l)
- Hidden files might require ls -a to discover

Real-World Relevance

This mimics real-world scenarios where:
- Sensitive credentials are stored in plaintext files
- Attackers gain access simply by exploring accessible directories
